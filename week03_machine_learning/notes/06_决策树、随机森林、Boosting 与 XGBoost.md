## 1. 决策树是什么

决策树通过一系列“如果……那么……”的分裂规则完成预测。

```text
根节点
├── 条件成立 → 左子树
└── 条件不成立 → 右子树
```

分类树的叶节点输出类别或类别概率；回归树的叶节点输出连续数值。

决策树根据**特征**和**阈值**逐步分裂数据。

优点：

- 规则直观；
- 不要求特征标准化；
- 可学习非线性关系和特征交互；
- 能同时处理不同尺度的数值特征。

缺点：

- 单棵树容易过拟合；
- 对训练数据扰动敏感；
- 深树通常高方差；
- 轴对齐分裂可能需要很多节点表示斜边界。

---

## 2. 决策树怎样学习

在每个节点，算法要决定：

1. 使用哪个特征；
2. 使用什么阈值；
3. 分裂后左右子节点如何划分样本。

目标是让分裂后的子节点比父节点更“纯”。

---

## 3. 纯度与熵

二分类节点中，设正类比例为 `p`，熵可写为：

```text
H(p) = -p log₂(p) - (1-p) log₂(1-p)
```

直觉：

- `p=0` 或 `p=1`：节点完全纯，熵为 0；
- `p=0.5`：两类最混杂，熵最大；
- 熵越大，类别越不确定。

需要注意：scikit-learn 决策树默认常使用 Gini impurity，而新版课程主要通过熵讲直觉。二者都是衡量节点混杂程度的方法。

---

## 4. 信息增益

信息增益衡量一次分裂减少了多少不纯度：

```text
信息增益
= 父节点熵
- 左子节点权重 × 左子节点熵
- 右子节点权重 × 右子节点熵
```

算法通常选择信息增益最大的分裂。

为什么对子节点熵加权？

```text
包含 90% 样本的子节点，影响应比只包含 10% 样本的子节点更大。
```

---

## 5. 停止分裂

树不应无限增长。常见停止条件：

- 已达到最大深度；
- 节点样本数太少；
- 继续分裂带来的增益很小；
- 叶节点样本数不能再减少；
- 节点已经足够纯。

scikit-learn 常用参数：

```python
DecisionTreeClassifier(
    max_depth=...,
    min_samples_split=...,
    min_samples_leaf=...,
    criterion="entropy" or "gini",
)
```

---

## 6. 分类特征与独热编码

树模型可以对数值阈值分裂，例如：

```text
radius_mean <= 14.5
```

分类特征不能总是直接当连续数字使用。若类别只是名称：

```text
red、green、blue
```

常用 One-Hot Encoding 转换为多个 0/1 特征。

本周数据集全部是连续数值特征，因此不需要额外独热编码。

---

## 7. 连续值特征

连续特征的候选分裂方式是寻找阈值：

```text
x_j <= t
```

算法会比较多个候选阈值，并选择带来最大不纯度下降的阈值。

---

## 8. 为什么单棵树容易高方差

训练数据发生少量变化时，最佳分裂可能改变：

```text
根节点改变
→ 后续整棵树结构都可能改变
```

因此单棵深树常表现为：

```text
训练 F1 很高
验证 F1 明显更低
```

`tree_ensemble_experiment.py` 会通过不同 `max_depth` 绘制训练/验证 F1，直观看到复杂度变化。

## 9. Bagging：多个模型并行平均

Bagging 的核心是：

```text
对训练集进行多次有放回抽样
→ 每个抽样集训练一个模型
→ 分类时投票，回归时平均
```

有放回抽样意味着同一个样本可能被抽到多次，也可能一次都没被抽到。

多个高方差模型的错误不完全相同，平均后可以降低整体方差。

---

## 10. 随机森林

随机森林在 Bagging 基础上又增加“特征随机性”。

每棵树：

1. 使用 Bootstrap 样本；
2. 每个节点只从随机特征子集中寻找最佳分裂；
3. 最后由多棵树投票。

为什么随机选择特征？

如果某个强特征总被选为根节点，所有树会非常相似。随机限制候选特征可以让树之间更有差异，从而让平均更有效。

### 10.1 重要参数

```python
RandomForestClassifier(
    n_estimators=300,
    max_depth=None,
    min_samples_leaf=1,
    max_features="sqrt",
    random_state=42,
    n_jobs=-1,
)
```

- `n_estimators`：树的数量；
- `max_depth`：单棵树最大深度；
- `max_features`：每次分裂考虑多少特征；
- `min_samples_leaf`：叶节点最少样本数；
- `n_jobs=-1`：使用多个 CPU 核心。

### 10.2 随机森林与单棵树

```text
单棵树：可解释、但不稳定、容易高方差
随机森林：更稳定、泛化通常更好、但整体更难解释
```

## 11. Boosting：顺序纠错式集成

Bagging 中各模型可以并行训练；Boosting 中模型通常按顺序训练。

核心直觉：

```text
第一个弱模型先预测
→ 后续模型更关注前面没有学好的部分
→ 多个弱模型加权组合
```

Boosting 往往可以得到很强的结构化数据性能，但参数不当时也会过拟合。

---

## 12. Gradient Boosting

Gradient Boosting 不是简单把“错误样本权重加大”，更一般地说，它让后续模型拟合当前损失函数的负梯度方向。

对平方误差回归，可以直观理解为：

```text
后续树学习前面模型的残差。
```

本项目使用 scikit-learn 的：

```python
GradientBoostingClassifier
```

它用于实践 Boosting 主线，但不是 XGBoost 的同义词。

---

## 13. XGBoost

XGBoost 是经过工程和正则化优化的梯度提升树实现。

常见特点：

- 对目标函数做二阶近似；
- 包含树复杂度正则化；
- 支持列采样和行采样；
- 高效并行与缺失值处理；
- 在结构化表格数据竞赛中非常常见。

重要参数：

```text
n_estimators：树数量
learning_rate：每棵树贡献步长
max_depth：单棵树深度
subsample：样本采样比例
colsample_bytree：特征采样比例
reg_lambda / reg_alpha：L2 / L1 正则化
```

XGBoost 需要额外安装：

```powershell
pip install xgboost
```

第一遍学习重点不是背参数，而是理解：

```text
它属于 Boosting；
树是顺序添加的；
每棵新树用于修正当前模型；
学习率和树数量共同控制模型容量。
```

## 14. Bagging 与 Boosting 对比

| 方面 | Bagging / Random Forest | Boosting / XGBoost |
|---|---|---|
| 训练关系 | 多个模型相对独立 | 模型按顺序纠错 |
| 主要目标 | 降低方差 | 降低偏差并提高拟合能力 |
| 并行性 | 较强 | 相对较弱 |
| 对噪声 | 通常较稳健 | 可能更关注难样本和噪声 |
| 关键参数 | 树数量、深度、特征采样 | 学习率、树数量、深度、采样与正则化 |

---

## 15. 特征重要性

树模型常提供 `feature_importances_`。

它反映特征在树分裂中带来的不纯度下降总量，但不能直接理解为因果关系。

局限：

- 高基数特征可能被偏爱；
- 相关特征会分摊重要性；
- 重要性不表示特征增加一定导致预测增加；
- 训练内置重要性可能与验证数据上的真实作用不同。

更可靠的扩展方法包括 permutation importance 和 SHAP。

---

## 16. 何时使用树模型

树和树集成特别适合：

- 表格型结构化数据；
- 非线性关系；
- 特征之间存在交互；
- 不希望手动标准化；
- 同时存在不同尺度特征。

神经网络通常更适合：

- 图像；
- 音频；
- 文本；
- 大规模非结构化数据；
- 需要端到端表示学习的任务。

这不是绝对规则，应通过验证集和业务约束比较。
