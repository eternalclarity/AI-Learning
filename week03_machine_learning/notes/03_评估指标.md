# 分类评估指标

> 新版课程对应：70～72（模型评估与数据划分）、75（基准性能）、79～80（开发迭代与错误分析）、85～86（倾斜数据集、Precision 与 Recall 权衡）。

---

## 1. 为什么不能只看 Accuracy

假设有 1000 个样本,**但样本中正类和父类的比例并不是50-50**：

```text
990 个负类
10 个正类
```

如果模型把所有样本都预测为负类：

```text
Accuracy = 990 / 1000 = 99%
```

但它一个正类都没有找出来。

**比如罕见病例样本分类，可能90%都是负类，10%是正类，那么模型只预测负类，仍有90% accuracy**

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-22 172100.jpg" alt="屏幕截图 2026-07-22 172100" style="zoom:50%;" />

这说明：

```text
高 Accuracy 不一定代表模型解决了真正重要的问题。
```

尤其在以下任务中更要谨慎：

- 欺诈检测；
- 故障检测；
- 安全告警；
- 风险筛查；
- 极少数事件识别。

### 1.1 多数类基线

在正式比较模型前，可以先计算一个最简单的基准：

```text
始终预测样本数量最多的类别。
```

如果一个复杂模型只比多数类基线高一点，说明模型可能没有学到足够有价值的规律。

但多数类基线只是最低参考，不等于业务可接受标准。新版课程中的“基准性能”还可能来自：

- 人类专家水平；
- 旧系统表现；
- 简单规则系统；
- 理论上可达到的性能；
- 业务规定的最低 Recall 或 Precision。

## 2. 先明确正类

所有二分类指标都依赖“谁是正类”。

本项目规定：

```text
1 = malignant（正类）
0 = benign（负类）
```

这样：

- Recall 表示真实恶性样本中有多少被识别出来；
- Precision 表示模型预测为恶性的样本中有多少确实是恶性；
- FN 表示恶性样本被漏判为良性。

如果交换正负类，同一组预测得到的 Precision、Recall、Specificity 会改变。

## 3. 混淆矩阵-Acc/Pre/Rec

**模型预测类型(predicted class) 和 实际类别(actual class) 2*2 构成的矩阵 称为 混淆矩阵：**

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-22 172957.jpg" alt="屏幕截图 2026-07-22 172957" style="zoom:50%;" />

二分类结果有四种情况。

| 名称 | 真实标签 | 预测标签 | 含义 |
|---|---:|---:|---|
| TP**（真阳性-True Positive)** | 1 | 1 | 正类被正确识别 |
| TN **(真阴性-True Negative)** | 0 | 0 | 负类被正确识别 |
| FP **(假阳性-False Positive)** | 0 | 1 | 负类被误报为正类 |
| FN **(假阴性-False Negative)** | 1 | 0 | 正类被漏报为负类 |

---

### 3.1 Accuracy -准确率

公式：

```text
Accuracy = (TP + TN) / (TP + TN + FP + FN)          %     T/ （T + F）
```

含义：

```text
全部样本(TP + TN + FP + FN) 中 预测正确(TP + TN) 的比例。
```

适用条件：

- 类别比例较均衡；
- 不同错误代价近似；
- 需要一个直观总体指标。

局限：

- 类别严重不均衡时可能误导；
- 不区分 FP 与 FN 的代价；
- 不反映概率排序能力。

---

### 3.2. Precision -精确率

公式：

```text
Precision = TP / (TP + FP)					  %  TP / P
```

中文常称：

- 精确率；
- 查准率。

它回答：

```text
模型预测为正类的样本(TP + FP) 中多少 真的为正类(TP) ？
```

Precision 高意味着 FP 少，也就是**误报**较少。

更重视 Precision 的例子：

- 推荐高成本人工审核对象；
- 自动封禁账号；
- 发送非常昂贵的干预措施；
- 搜索结果需要高相关性。

---

### 3.3. Recall -召回率

公式：

```text
Recall = TP / (TP + FN)					%   TP / (Actual P)
```

中文常称：

- 召回率；
- 查全率；
- 灵敏度；
- True Positive Rate。

它回答：

```text
所有真实正类(TP + FN) 中有多少 被模型找到(TP)？
```

Recall 高意味着 FN 少，也就是**漏报**较少。

更重视 Recall 的例子：

- 风险筛查；
- 火灾告警；
- 欺诈初筛；
- 故障预警；
- 安全漏洞扫描。

### 3.4. Specificity -特异度

公式：

```text
Specificity = TN / (TN + FP)				%   TN / (Actual N)
```

中文常称：

- 特异度；
- 真负率。

它回答：

```text
所有真实负类(TN + FP) 中有多少 被正确识别为负类(TN) ？
```

Recall 关注正类，Specificity 关注负类。

## 4. 分类阈值

默认阈值 0.5 只是常见起点，不是所有任务的最优选择。阈值应当根据验证集和业务代价确定。

严格规则：

```text
模型参数在训练集学习；
模型和阈值在验证集选择；
测试集只做最终一次评估。
```

默认阈值常为 0.5：

```text
h(x)/p ≥ 0.5 → 预测 1 -> 正类
h(x)/p < 0.5 → 预测 0 -> 负类
```

> **Precision 与 Recall 之间经常存在权衡,高pre往往意味着低Rec**

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-23 085055.jpg" alt="屏幕截图 2026-07-23 085055" style="zoom: 50%;" />

**提高阈值**通常会：

- 预测更少正类；
- Precision 可能上升；
- Recall 下降。
- FN 增加；

**降低阈值**通常会：

- 预测更多正类；
- Precision 可能下降
- Recall 上升；
- FP 增加；

## 8. F1

**F1 是 Precision 与 Recall 的调和平均数，或者说1/p和1/R的平均数取倒数**。公式：

```text
F1 = 2 × Precision × Recall / (Precision + Recall)
```

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-23 090401.jpg" alt="屏幕截图 2026-07-23 090401" style="zoom:50%;" />

为什么不用普通算术平均？

**因为调和平均会更强地惩罚其中一个指标特别低的情况，他会给P和R中较小值更大的权重。**

例如：

```text
Precision = 1.0
Recall = 0.1
```

算术平均是 0.55，但 F1 只有约 0.18，更能反映模型不平衡的问题。

适合：

- 希望 Precision 与 Recall 都不能太差；
- 类别不完全均衡；
- FP 与 FN 都值得关注。

局限：

- 不包含 TN；
- 默认认为 Precision 与 Recall 同等重要；
- 仍然依赖固定阈值。

---

## 9. F-beta

更一般的形式：

```text
Fβ = (1 + β²)PR / (β²P + R)
```

- β > 1：更重视 Recall；
- β < 1：更重视 Precision；
- β = 1：就是 F1。

如果漏报代价明显高于误报，可以考虑 F2。

---

## 10. Balanced Accuracy

Balanced Accuracy 通常是不同类别召回率的平均。

二分类时：

```text
Balanced Accuracy = (Recall + Specificity) / 2
```

它在类别不均衡时比普通 Accuracy 更公平，因为不会让多数类完全支配结果。

---

## 12. ROC 曲线

ROC 曲线横轴：

```text
False Positive Rate = FP / (FP + TN)
```

纵轴：

```text
True Positive Rate = Recall = TP / (TP + FN)
```

ROC 曲线通过改变分类阈值，观察模型在不同阈值下的表现。

理想模型希望：

```text
FPR 尽量低
TPR 尽量高
```

因此曲线越靠近左上角通常越好。

---

## 13. ROC-AUC

AUC 是 ROC 曲线下面积。

常见直觉：

```text
AUC = 随机抽取一个正样本和一个负样本时，
模型把正样本排在负样本前面的概率能力。
```

范围通常在 0 到 1：

- 1.0：完美排序；
- 0.5：接近随机；
- 小于 0.5：可能标签方向或分数方向相反。

优点：

- 不依赖单一阈值；
- 衡量排序能力；
- 便于比较不同模型。

局限：

- 类别极度不平衡时，ROC-AUC 可能看起来仍很好；
- 不直接告诉你实际阈值下有多少误报和漏报；
- 不能代替混淆矩阵和业务成本分析。

---

## 14. PR 曲线与 Average Precision

Precision-Recall 曲线横纵轴直接关注：

- Recall；
- Precision。

在正类非常稀少时，PR 曲线经常比 ROC 曲线更有解释价值。

Average Precision 用一个数概括 PR 曲线表现。

本周代码主要要求 ROC-AUC，但你需要知道：

```text
正类极少时，应考虑同时查看 PR 曲线和 AP。
```

## 15. Macro、Micro 与 Weighted

多分类中常见三种平均方式。

### 15.1 Macro

先分别计算每个类别指标，再直接平均。

每个类别权重相同，少数类也会被重视。

### 15.2 Weighted

按每个类别样本数加权平均。

多数类影响更大。

### 15.3 Micro

先汇总所有类别的 TP、FP、FN，再计算指标。

更接近总体样本层面的表现。

本周是二分类，并明确 `pos_label=1`，所以主要使用 binary 指标。

## 16. 指标选择必须结合任务代价

没有一个指标永远最好。

选择指标前应回答：

1. 谁是正类？
2. FP 的代价是什么？
3. FN 的代价是什么？
4. 类别是否不平衡？
5. 是否需要概率排序？
6. 最终阈值怎样确定？

本项目选择验证集 F1 作为主要指标，是为了同时兼顾 Precision 与 Recall。

但在真实高风险筛查中，可能更强调 Recall，并设置最低 Recall 约束，再在满足约束的模型中提高 Precision。

---

## 17. 指标之后还要做错误分析

假设两个模型的 F1 都是 0.94，它们的实际问题可能完全不同：

- 模型 A 主要发生 FN，也就是漏掉正类；
- 模型 B 主要发生 FP，也就是误报负类；
- 模型 C 的错误集中在某一类边界样本；
- 模型 D 的错误集中在特征缺失或噪声较大的样本。

因此应保存逐样本预测结果，并检查：

```text
哪些样本预测错了？
错误属于 FP 还是 FN？
是否集中在某种数据子群？
是数据质量问题、特征问题还是模型问题？
```

本项目的 `evaluate_best_model.py` 会生成 `test_predictions.csv`，用于后续错误分析。

`threshold_tradeoff_experiment.py` 则只在验证集上扫描阈值，生成 Precision、Recall 和 F1 随阈值变化的曲线；它不会使用测试集。

## 18. 一个完整示例

假设：

```text
TP = 30
TN = 50
FP = 10
FN = 10
```

总样本：

```text
100
```

计算：

```text
Accuracy = (30 + 50) / 100 = 0.80
Precision = 30 / (30 + 10) = 0.75
Recall = 30 / (30 + 10) = 0.75
Specificity = 50 / (50 + 10) ≈ 0.8333
F1 = 0.75
```

不要只记公式，要能够解释每个分母代表什么集合。

```python
accuracy_score(y_true, y_pred)
precision_score(y_true, y_pred, pos_label=1)
recall_score(y_true, y_pred, pos_label=1)
f1_score(y_true, y_pred, pos_label=1)
roc_auc_score(y_true, y_score)

```

注意：

```text
Accuracy、Precision、Recall、F1 使用离散预测 y_pred
ROC-AUC 使用连续分数 y_score
```

错误做法：

```python
roc_auc_score(y_true, y_pred)
```

虽然某些情况下代码能运行，但会丢失不同阈值下的排序信息。

3. FN 增加会直接降低哪个指标？

   > 直接降低 **Recall（召回率）**。
   >
   > 因为：
   >
   > Recall=TPTP+FNRecall=\frac{TP}{TP+FN}Recall=TP+FNTP
   >
   > 当 TP 不变、FN 增加时，分母变大，Recall 就会下降。
   >
   > FN 表示真实正类被模型漏掉，所以 FN 越多，模型找出的正类越不完整。
   >
   > 例如有 100 个恶性患者：
   >
   > - 找出 90 个，FN 为 10，Recall 为 90%；
   > - 只找出 70 个，FN 为 30，Recall 为 70%。
   >
   > 因此，在疾病筛查中通常特别关注 Recall，因为漏掉真正患者的后果可能很严重。

4. FP 增加会直接降低哪个指标？

   > 会直接降低 **Precision（精确率）**。
   >
   > 因为：
   >
   > Precision=TPTP+FPPrecision=\frac{TP}{TP+FP}Precision=TP+FPTP
   >
   > 当 TP 不变、FP 增加时，分母变大，Precision 就会下降。
   >
   > FP 表示模型预测为正，但实际并不是正类。FP 越多，说明模型的“正类预测”中混入了越多错误结果。
   >
   > 例如模型预测 100 人为恶性：
   >
   > - 其中 90 人确实恶性，Precision 为 90%；
   > - 其中只有 60 人确实恶性，Precision 为 60%。

6. 为什么 F1 不考虑 TN？

   > 1 是 Precision 和 Recall 的调和平均数：
   >
   > F1=2⋅Precision⋅RecallPrecision+RecallF1=2\cdot\frac{Precision\cdot Recall}{Precision+Recall}F1=2⋅Precision+RecallPrecision⋅Recall
   >
   > 也可以写成：
   >
   > F1=2TP2TP+FP+FNF1=\frac{2TP}{2TP+FP+FN}F1=2TP+FP+FN2TP
   >
   > 其中没有 TN，是因为 F1 专门关注**正类是否预测得准确且完整**。
   >
   > 它关心：
   >
   > - 找出来的正类有多少是真的；
   > - 真实正类有多少被找出来了。
   >
   > TN 表示负类被正确识别，但它不能说明正类识别得怎么样。
   >
   > 在负类数量特别多的任务中，如果将 TN 纳入指标，大量 TN 可能掩盖模型对正类识别能力不足的问题。
   >
   > 例如疾病筛查中，即使识别对了很多健康人，也不能弥补漏掉大量恶性患者的问题。
   >
   > 因此：
   >
   > > F1 是一个以正类为中心的评价指标，不是一个全面衡量正类与负类的指标。
