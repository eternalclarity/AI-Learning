## 1. 拟合

训练模型的过程，是让模型根据训练数据形成从特征 `X` 到标签 `y` 的规律。

理想状态是：

```text
既能较好解释训练数据，
又能推广到未见过的新数据。
```

只看训练集无法判断模型是否真的学到了可推广规律，因此必须结合训练集、验证集和测试集。

## 2. 欠拟合-高偏差(bias)

欠拟合表示模型连训练数据中的主要规律都没有学好。

常见表现：

```text
训练集误差较高；
验证集误差也较高；
两者差距通常不大。
```

“高偏差”强调模型存在系统性不足，例如：

- 模型过于简单； - （模型阶数太低/特征太少/表达能力低）
- 特征表达能力不足；
- 正则化太强；  -> λ 值设置的太大，所有特征的权重几乎为0
- 训练没有充分收敛；
- 优化设置不合理；
- 任务本身需要非线性模型，但使用了过于简单的线性边界。

### 2.1 可能的改进方向

- **增加有意义的特征；**
- 使用多项式或交互特征；
- **使用表达能力更强的模型；**
- **减弱正则化；**
- 延长训练或改进优化器；
- 检查代码和标签是否正确。

注意：

> **(欠拟合）高偏差时，盲目增加更多同分布训练数据通常帮助有限。**
>
> **因为模型本身连已有数据都没有学好。** -> 7.偏差与方差

## 3. 过拟合与高方差(variance)

过拟合表示模型对训练集表现很好，但在新数据上明显下降。

常见表现：

```text
训练集误差很低；
验证集误差明显更高；
训练和验证之间存在较大泛化差距。
```

高方差常见原因：

- 模型过于复杂； -> （高阶模型/特征太多/表达能力太强） / λ = 0
- 训练样本过少；
- 特征数量相对样本数过多；
- 噪声被模型当成规律；
- 超参数反复针对同一验证集调整；
- 数据划分存在偶然性；
- 训练集与实际应用数据分布不一致。

### 3.1 可能的改进方向

- **增加更多高质量训练数据；**

- **加强 L1/L2 正则化；**

- 降低决策树深度；

- **减少不必要特征；**

- **使用数据增强；**

- 使用 Bagging、随机森林等降低方差；

- 使用交叉验证检查稳定性；

- 使用 Early Stopping；

- 检查训练和部署数据分布是否一致。

## 4. 偏差—方差不是两个独立开关

模型复杂度增加时，常出现：

```text
偏差下降，方差上升。
```

模型复杂度降低时，常出现：

```text
方差下降，偏差上升。
```

但机器学习实践不是永远只能二选一。更多高质量数据、更合理特征、集成学习和更好的优化，有时可以同时改善二者。

## 5. 训练集、验证集与测试集

**训练集 -> 拟合参数**

**交叉验证集 -> 选择模型**

**测试机 -> 最终的评估指标**

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-22 100757.jpg" alt="屏幕截图 2026-07-22 100757" style="zoom: 50%;" />

### 5.1 训练集

用途：

```text
学习模型参数和预处理参数。
```

例如：

- 逻辑回归权重；
- 决策树分裂规则；
- StandardScaler 的均值和标准差；
- 神经网络权重。

### 5.2 验证集 / 交叉验证集

用途：

```text
比较模型、选择超参数、选择分类阈值、决定下一步实验。
```

本项目使用单独验证集进行初学阶段的清晰演示，并在学习曲线实验中使用分层交叉验证。

### 5.3 测试集

用途：

```text
模型与所有超参数确定后，进行一次最终独立评估。
```

测试集不能用于：

- 选择模型；
- 选择阈值；
- 决定正则化强度；
- 决定树深度；
- 反复观察后继续修改方案。

如果反复根据测试集结果修改模型，测试集就已经变成了验证集。

## 6. 建立基准性能水平

判断训练表现是否“低”，不能只依靠固定阈值，例如：

```text
F1 < 0.85 就一定是高偏差
```

这种判断不可靠，因为不同任务难度不同。

应该先建立基准：

- **人类专家水平；**

- 旧系统表现；

- 简单规则模型；

- 多数类基线；

- 理论最优或可达到性能；

- 业务最低要求。

  <img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-22 110402.jpg" alt="屏幕截图 2026-07-22 110402" style="zoom:50%;" />

### 6.1 可避免偏差

假设：

```text
基准 F1 = 0.98
训练 F1 = 0.86
验证 F1 = 0.84
```

训练性能距离基准较远，说明可能存在较大可避免偏差。

若：

```text
基准 F1 = 0.98
训练 F1 = 0.97
验证 F1 = 0.84
```

训练表现接近基准，但验证明显下降，更像高方差。

### 6.2 本项目为什么不自动断言

`bias_variance_experiment.py` 不再使用固定分数阈值直接宣布“高偏差”或“高方差”。它会输出：

- 训练 F1；
- 交叉验证 F1；
- 泛化差距；
- 可选基准性能差距；
- 需要结合完整曲线判断的提示。

这是比硬编码阈值更规范的做法。

## 8.正则化

正则化:缓解模型特征是否过多，特征权重是否过大.

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-21 103848.jpg" alt="屏幕截图 2026-07-21 103848" style="zoom:80%;" />

### 8.1 L1 正则化

L1 惩罚权重绝对值：

```text
Loss = 原损失 + λ/m × Σ |w_j|
```

特点：

- 倾向于产生稀疏权重；
- 部分权重可能变为 0；
- 可起到一定特征选择作用；
- 在强相关特征中选择可能不稳定。

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-21 104356.jpg" alt="屏幕截图 2026-07-21 104356" style="zoom:80%;" />

当某个特征对应的权重变成 0：

​									wjxj = 0

这个特征就不再影响预测，相当于被模型自动删除。

因此：

> **L1 正则化具有稀疏化和自动特征选择的效果。**

#### 8.1.1 PyTorch 写法

```
# 原始预测损失
original_loss = criterion(outputs, labels)

# 计算所有权重绝对值之和
l1_penalty = sum(
    parameter.abs().sum()
    for parameter in model.parameters()
)

# 总损失 = 原始损失 + L1 惩罚
loss = original_loss + l1_lambda * l1_penalty
```



### 8.2. L2 正则化

L2 正则化向**目标函数(损失函数J/loss）**加入**权重平方惩罚**：

```text
Loss = 原损失 + λ/(2m) × Σ w_j²
```

作用：

- 限制权重过大；
- 降低模型对少数特征的极端依赖；
- 常用于缓解高方差；
- 通常不会把大量权重精确压到 0。

#### 8.2.1 PyTorch 写法

通常使用 AdamW：

```
optimizer = torch.optim.AdamW(

    weight_decay=1e-4,

```

#### 8.3 Dropout

Dropout 不直接限制权重，而是在训练过程中：

> **每次前向传播，随机把一部分神经元的输出暂时置为 0。**

例如某一层输出为：

```
[0.8, 1.2, 0.5, 2.0, 0.3]
```

使用：

```
nn.Dropout(p=0.2)
```

某一次前向传播可能变成：

```
[0.8, 0, 0.5, 2.0, 0.3]
```

下一次又可能变成：

```
[0.8, 1.2, 0.5, 0, 0.3]
```

被丢弃的位置每次都可能不同。

##### Dropout 的目的

没有 Dropout 时，神经元之间可能形成固定搭配：

```
神经元 A 总是依赖神经元 B
神经元 B 总是依赖神经元 C
```

模型可能只会使用一套固定套路。

加入 Dropout 后，某个神经元随时可能被临时关闭，其他神经元必须学会独立完成任务。

因此：

> **Dropout 防止神经元之间形成过度依赖。**

```text
L1
  ↓
限制模型权重
  ↓
让部分权重变成 0

L2
  ↓
限制模型权重
  ↓
让整体权重不要过大

Dropout
  ↓
限制神经元之间的依赖
  ↓
训练时随机关闭部分输出
```

最精炼的区别是：

> **L1 删除不重要的连接，L2削弱过强的连接，Dropout 打乱神经元之间的固定合作。**

## 10. 决策树如何控制过拟合

树模型没有神经网络那样的 L2 权重，但可以通过结构超参数控制复杂度：

- `max_depth`：最大深度；
- `min_samples_split`：节点继续分裂所需最小样本数；
- `min_samples_leaf`：叶节点最少样本数；
- `max_features`：每次分裂允许考虑的特征数量；
- 剪枝参数。

单棵树深度过大时，往往训练分数接近 1，但验证分数下降，是很直观的高方差案例。

`tree_ensemble_experiment.py` 会绘制不同 `max_depth` 下的训练和验证 F1。

## 12. Early Stopping

Early Stopping 监控验证集性能：

```text
验证性能持续不改善
→ 停止训练
→ 恢复最佳阶段参数
```

它既节省计算，也可以减少训练后期对训练数据的过度拟合。

本项目的 `MLPClassifier` 使用：

```python
early_stopping=True
```

但要注意：它会从传入 `fit()` 的训练数据内部再划分一小部分作为早停验证数据。

## 13. 交叉验证

K 折交叉验证将开发数据分成 K 份：

```text
每次用 K-1 份训练，1 份验证；
重复 K 次；
最后汇总平均值和波动。
```

分类任务通常使用 `StratifiedKFold`，以保持每折类别比例近似一致。

优点：

- 降低单次划分的偶然性；
- 更稳定地比较超参数；
- 适合样本量较小的数据集。

缺点：

- 训练成本增加约 K 倍；
- 如果预处理不放进 Pipeline，仍可能产生泄漏；
- 不能取代最终独立测试集。

## 14. 数据泄漏

数据泄漏是指训练过程中获得了本不应提前知道的信息。

### 14.1 先对全部数据标准化

错误：

```python
X_scaled = scaler.fit_transform(X)
X_train, X_test = train_test_split(X_scaled)
```

正确：

```text
先划分数据；
StandardScaler 只在训练集 fit；
验证集和测试集只 transform。
```

最安全方式是使用 Pipeline。

### 14.2 使用测试集调参

错误：

```text
在测试集上比较多个模型，选择最好的一个。
```

这使测试集参与了模型选择。

### 14.3 重复样本跨集合

如果同一对象的高度相似样本同时出现在训练集和测试集，结果可能虚高。

### 14.4 时间或用户泄漏

时间序列和用户数据不能总是随机拆分。例如同一用户的记录分布到训练和测试两边，可能让模型间接记住用户。

## 16. 错误分析

平均指标只能告诉你总体表现，不能告诉你模型为什么错。

建议建立错误分析表：

| 样本 | 真实标签 | 预测标签 | 错误类型 | 可能原因 |
|---|---:|---:|---|---|
| 样本A | 1 | 0 | FN | 边界样本、特征异常 |
| 样本B | 0 | 1 | FP | 与正类特征相似 |

检查：

- 错误是否集中在某些类别；
- 是否存在标签错误；
- 是否缺少关键特征；
- 是否存在数据质量问题；
- 模型是否在某个子群体上明显更差。

## 17. "增加数据"

“增加数据”可能包括：

- 收集更多真实样本；

- 针对错误类型定向收集；

- 修正标签；

- 改善数据一致性；

- **对于图片和语音识别尤其可以使用数据增强**；

## 18. 迁移学习-机器学习最后归宿

欢迎快进到迁移学习！

迁移学习**先使用在其他任务上学到的表示，再针对当前任务微调**。 **- 把模型在别的任务中学到的知识，迁移到新任务中继续使用**

> **迁移学习，先通过(有大量数据集)(相似的不同任务)（如识别猫，狗）训练好的模型参数直接copy，同时清空（最后输出层）的参数，或（用自己的输出层替换掉别人训练好的模型的输出层），相当于有了那个模型的基础作为起点，最后只需(少量数据集)微调最后一层参数即可。**比如这个图像分类任务，先通过预训练让模型学会看图能力，再用较小的数据集进行微调，就可以解决图像分类任务。

<img src="D:\code.py\workspace\AI-Learning\week03_machine_learning\notes\img\屏幕截图 2026-07-22 162852.jpg" alt="屏幕截图 2026-07-22 162852" style="zoom:50%;" />

适合：

- 当前任务标注数据少；
- 存在相关的大规模预训练模型；
- 输入类型相似，例如图像、文本或语音。

>例子：
>
>假设已经有一个在 ImageNet 上训练好的图像分类模型，它已经学会了：
>
>- 边缘
>- 纹理
>- 形状
>- 物体结构
>
>现在你要训练 FashionMNIST 衣服分类模型，就没必要从随机参数开始，可以直接使用预训练模型的参数，再针对衣服分类任务进行训练。
>
>这就是迁移学习：
>
>> 把模型在旧任务中学到的知识，迁移到新任务中继续使用。
>
>典型流程是：
>
>```
>预训练模型
>    ↓
>保留已有参数
>    ↓
>修改最后的分类层
>    ↓
>在新数据集上微调
>```
>
>例如：
>
>```
>model = torchvision.models.resnet18(weights="DEFAULT")
>
># 替换最后的分类层
>model.fc = nn.Linear(
>    model.fc.in_features,
>    10
>)
>```
>
>这里 ResNet 原本可能识别 1000 类物体，现在把最后一层改成 10 类，再用自己的数据继续训练。



这部分将在后续 Transformer 和大模型微调中再次出现。

> ## GPT 与迁移学习：这是 GPT 最核心的训练思想之一
>
> 最早的 GPT 就采用了非常典型的两阶段训练：
>
> > 大规模通用文本预训练 → 在具体任务数据上微调
>
> OpenAI 在 GPT-1 的介绍中明确说明：先在大量文本上进行无监督预训练，让模型获得通用语言能力；再使用较小的监督数据集进行微调，使模型解决具体任务。
>
> 这本质上就是**迁移学习**：
>
> ```
> 预训练阶段：
> 海量通用文本
>     ↓
> 学会语言、知识、语义、推理模式
>     ↓
> 通用 GPT 基础模型
> 
> 迁移阶段：
> 通用 GPT 基础模型
>     + 指令数据 / 对话数据 / 任务数据
>     ↓
> 适合聊天、问答、代码等任务的模型
> ```
>
> 例如，模型预训练时已经学会了：
>
> - 中文和英文的语言规律
> - 基本世界知识
> - 代码结构
> - 一些推理模式
>
> 之后不需要针对每个任务都从随机参数重新训练，只需要通过微调或其他后训练方式，把这些通用能力迁移到：
>
> - 对话问答
> - 代码生成
> - 文本摘要
> - 客服系统
> - 医疗、法律等垂直领域
>
> 现代 ChatGPT 的完整训练过程通常可以概括为：
>
> ```
> 预训练
>   ↓
> 监督微调 SFT
>   ↓
> 基于人类反馈的强化学习等后训练
>   ↓
> 适合对话和指令遵循的模型
> ```
>
> OpenAI 对 ChatGPT 基础模型的公开说明也将其开发过程分为预训练和后训练阶段；InstructGPT 则是在预训练 GPT-3 的基础上，通过人类示范数据和 RLHF 进一步训练。
>
> 因此，可以明确地说：
>
> > **GPT 的“预训练基础模型 + 后续微调”就是迁移学习思想的典型应用。**

## 19. 机器学习项目完整周期

```text
确定范围和目标

→ 训练初始系统
```

- 反馈闭环；
- 模型版本和可复现性。

## 20. 公平性、偏见与伦理

模型总体指标高，不代表对所有群体都同样可靠。

需要考虑：

- 数据是否代表真实使用人群；
- 标签是否带有人为偏见；
- 不同子群体的错误率是否差异明显；
- 模型错误由谁承担后果；
- 是否提供人工复核和申诉机制；
- 是否清楚说明模型用途与限制。

本项目是教学实验，不能用于真实医疗判断。
