## 1. 机器学习到底是什么

机器学习不是“让计算机像人一样思考”的神秘技术，而是：

```text
让计算机从数据中寻找可用于预测或决策的规律。
```

传统编程通常是：

```text
人工规则 + 输入数据 → 输出结果
```

机器学习通常是：

```text
训练数据 + 正确答案 → 学习算法 → 模型
模型 + 新数据 → 预测结果
```

例如，**人工判断邮件是否为垃圾邮件时**，可能编写很多规则：

- 标题是否包含“免费”；
- 是否有大量链接；
- 发件人是否陌生。

机器学习则可以输入大量“邮件特征 + 是否垃圾邮件”的样本，让算法自己学习特征与标签之间的关系。

## 2. 吴恩达给出的经典定义

课程中常引用两种定义。

### 2.1 Arthur Samuel 的定义

核心思想是：

> 让计算机在没有被明确编程的情况下获得学习能力。

重点不是完全没有程序，而是程序员不再逐条写出所有判断规则，而是写出学习算法，让算法从数据中形成模型。

### 2.2 Tom Mitchell 的定义

可以概括为：

```text
如果一个程序在任务 T 上的表现 P，随着经验 E 的增加而提高，
就说该程序从经验 E 中学习了任务 T。
```

例如垃圾邮件分类：

- T：判断邮件是不是垃圾邮件；
- E：已经标注好的历史邮件；
- P：分类准确率、Precision、Recall 或 F1。

这个定义提醒我们：讨论一个机器学习问题时，必须明确三件事：

1. 做什么任务；
2. 用什么数据学习；
3. 用什么指标判断是否变好。

## 3. 监督学习

监督学习的数据包含：

```text
输入特征 X + 正确标签 y
```

模型通过训练样本学习从 X 到 y 的映射：

```text
f: X → y
```

> 类似于第二周做的图像分类，给了特征images和标签labels，通过二者的映射关系来调整参数。
>
> 监督 -> labels.

### 3.1 基本术语

假设有 m 个训练样本，每个样本有 n 个特征。

- `m`：样本数量；
- `n`：特征数量；
- `x^(i)`：第 i 个样本的输入特征；
- `y^(i)`：第 i 个样本的真实标签；
- `(x^(i), y^(i))`：第 i 条完整训练样本；
- `X`：所有输入特征组成的矩阵；
- `y`：所有标签组成的向量；
- `f` 或 `hθ`：模型或假设函数；
- `θ`、`w`、`b`：模型参数。

在本周数据集中：

```text
m = 569
n = 30
X.shape = [569, 30]
y.shape = [569]
```

每一行代表一个样本，每一列代表一个数值特征。

## 4. 分类与回归

监督学习通常分成两大类。

### 4.1 回归 -> 线性回归模型

回归预测连续数值：

- 房价；
- 温度；
- 销售额；
- 设备剩余寿命。

输出可以是实数轴上的任意值：

```text
120.5
36.8
800000
```

第二周手写的线性回归就是回归任务。

### 4.2 分类

分类预测离散类别：

- 正常 / 异常；
- 垃圾邮件 / 正常邮件；
- 猫 / 狗；
- 良性 / 恶性。

**二分类**常用标签：

```text
0 = 负类
1 = 正类
```

本周项目规定：

```text
0 = benign
1 = malignant
```

这个规定不是数学强制要求，而是为了让 Precision、Recall 等指标中的“正类”具有明确业务含义。

## 5. 无监督学习

无监督学习只有输入 X，没有人工提供的标签 y。

```text
只有数据，没有标准答案
```

> 只提供特征，不提供标签，代表着不知道答案，以及谁是谁。
>
> 依靠聚类算法把相互靠的近的一群特征聚成一类。

常见任务：

### 5.1 聚类

**自动把相似样本分组**，例如：

- 用户画像；
- **新闻主题聚类 -> 不同的新闻报道可以根据相同的报道主题聚成一个个类别**；
- 商品分群；
- 基因表达样本分组。

### 5.2 降维

把高维数据压缩到更低维，尽量保留重要结构，例如 PCA。

用途包括：

- 可视化；
- 数据压缩；
- 降低噪声；
- 减少计算量。

### 5.3 异常检测

学习“正常数据”的分布，再发现偏离正常分布的样本。

## 6. 半监督、自监督与强化学习

虽然本周不深入，但需要知道它们的位置。

### 6.1 半监督学习

同时使用：

- 少量有标签数据；
- 大量无标签数据。

适合人工标注昂贵的场景。

### 6.2 自监督学习

从原始数据本身构造训练目标。

例如大语言模型通过预测下一个 token 学习语言规律。

### 6.3 强化学习

智能体与环境交互，根据奖励学习行动策略。

核心元素：

- 状态；
- 动作；
- 奖励；
- 策略。

## 7. 模型、参数与超参数

### 7.1 模型

模型是从输入到输出的函数结构。

线性回归：

```text
f(x) = wx + b
```

逻辑回归：

```text
f(x) = sigmoid(wᵀx + b)
```

神经网络则由多层线性变换和非线性激活组成。

### 7.2 参数

参数由训练数据学习：

- 权重 w；
- 偏置 b；
- 决策树分裂规则；
- 神经网络权重。

调用：

```python
model.fit(X_train, y_train)
```

就是让模型根据训练集学习参数。

### 7.3 超参数

超参数由人预先设置，或者通过验证集、交叉验证选择：

- 学习率；
- 正则化强度；
- SVM 的 C 与 gamma；
- 随机森林树的数量；
- 最大深度；
- MLP 隐藏层大小。

参数与超参数的区别：

```text
参数：训练过程中学出来
超参数：训练开始前设置或通过模型选择确定
```

## 8. 损失函数与代价函数

模型需要一个数字衡量预测有多差。

### 8.1 单样本损失

损失函数衡量一条样本的预测误差。

### 8.2 整体代价

代价函数通常表示训练集上损失的平均值。

吴恩达课程常使用：

```text
J(θ)
```

表示参数 θ 对应的**整体代价**。

训练的目标是：

```text
找到一组参数，使 J(θ) 尽可能小。
```

注意：

```text
训练损失小 ≠ 对未知数据一定好
```

如果模型只记住训练数据，也可能出现训练损失很小、验证损失很大，这就是**过拟合**的重要表现。

## 9. 梯度下降

> 计算参数不只有梯度下降法，还有正规方程法这种纯数学直接计算参数
>
> 但是这种方法需要计算（xT ^ x) -1，计算逆适用数据量小，太大的话花费的时间是非常巨大的
>
> 所以一般数据集比较大都用 梯度下降法。

**梯度表示代价函数在当前位置变化最快的方向**。

梯度下降更新形式：

```text
参数 := 参数 - 学习率 × 梯度
```

学习率 α 控制每一步的大小。

### 9.1 学习率太小

- 收敛慢；
- 需要更多迭代。

### 9.2 学习率太大

- 可能在最低点附近震荡；
- 可能发散；
- 损失可能越来越大。

### 9.3 第二周与第三周的联系

第二周 PyTorch 中：

```python
loss.backward()
optimizer.step()
```

第三周 scikit-learn 中：

```python
model.fit(X_train, y_train)
```

`fit()` 内部替我们完成了相应的优化流程，因此代码更短，但原理并没有消失。

## 10. 特征缩放

**不同特征的数值范围可能相差很大**。

例如：

```text
特征 A：0.01 到 0.10
特征 B：100 到 1000
```

如果模型依赖距离或梯度优化，**大尺度特征可能主导计算**。

StandardScaler 执行：

```text
z = (x - μ) / σ
```

其中：

- μ：训练集该特征的均值；
- σ：训练集该特征的标准差。

标准化后通常：

```text
均值约为 0
标准差约为 1
```

需要特别注意：

```text
μ 和 σ 只能由训练集计算
```

不能先对全部数据 `fit_transform()` 再划分，否则验证集和测试集的信息会泄漏到训练过程。

## 12. 传统机器学习与深度学习的区别

| 方面 | 传统机器学习 | 深度学习 |
|---|---|---|
| 常见模型 | 逻辑回归、SVM、随机森林 | CNN、RNN、Transformer |
| 特征 | 常需要人工设计或结构化特征 | 可自动学习多层表示 |
| 数据规模 | 小中型数据常表现很好 | 通常更依赖大量数据 |
| 计算资源 | CPU 经常足够 | 经常需要 GPU |
| 可解释性 | 部分模型相对容易解释 | 通常更复杂 |
| 训练方式 | 常用 `fit()` 封装 | 常自己写训练循环 |

不是说深度学习一定优于传统机器学习。

对于：

- 数据量较小；
- 特征已经结构化；
- 需要快速实验；
- 需要一定可解释性；

传统机器学习往往是很强的基线。

2. 监督学习为什么需要 y？

   > 监督学习的数据通常表示为：
   >
   > (x,y)(x, y)(x,y)
   >
   > 其中：
   >
   > - xxx：输入特征；
   > - yyy：正确答案，也叫标签或目标值。
   >
   > 例如：
   >
   > ```
   > 图片 x → 模型预测 → 预测结果 ŷ
   > 图片对应的真实类别 → y
   > ```
   >
   > 模型需要将预测值 y^\hat{y}y^ 和真实值 yyy 进行比较，才能计算损失：
   >
   > Loss(y^,y)Loss(\hat{y}, y)Loss(y^,y)
   >
   > 然后根据损失进行反向传播，修改模型参数。
   >
   > 因此，`y` 相当于给模型提供了“标准答案”。没有 `y`，模型就不知道自己的预测是对还是错，也无法直接按照监督学习的方式计算误差。
   >
   > 例如：
   >
   > ```
   > outputs = model(images)          # 模型预测值 ŷ
   > loss = criterion(outputs, labels)  # labels 就是 y
   > ```
   >
   > 不过，没有 `y` 并不意味着无法进行机器学习，还可以使用：
   >
   > - 无监督学习；
   > - 自监督学习；
   > - 强化学习。
   >
   > 只是它们获得学习信号的方式不同。

3. 分类和回归的输出有什么区别？

   > #### 分类任务
   >
   > 分类任务输出的是一个**离散类别**。
   >
   > 例如：
   >
   > ```
   > 猫
   > 狗
   > 鸟
   > ```
   >
   > 或者：
   >
   > ```
   > 0、1、2
   > ```
   >
   > 神经网络通常不会直接输出“猫”或“狗”，而是先输出每个类别的分数：
   >
   > ```
   > 猫：2.1
   > 狗：5.3
   > 鸟：0.8
   > ```
   >
   > 然后选择分数最大的类别：
   >
   > ```
   > 预测结果：狗
   > ```
   >
   > 分类结果的可能取值通常是有限的。
   >
   > #### 回归任务
   >
   > 回归任务输出的是一个**连续数值**。
   >
   > 例如：
   >
   > - 房价：325,800 元；
   > - 温度：26.7℃；
   > - 销量：1543.2；
   > - 身高：178.5 cm。
   >
   > 回归的输出可以是某个范围内的任意实数。
   >
   > #### 核心区别
   >
   > | 类型 | 输出     | 示例             |
   > | ---- | -------- | ---------------- |
   > | 分类 | 离散类别 | 猫、狗、鸟       |
   > | 回归 | 连续数值 | 房价、温度、身高 |

6. 为什么训练损失很低仍可能效果不好？

   > 训练损失低，只说明模型对**训练集**拟合得好，不代表它对新数据也表现好。
   >
   > 最常见的原因是**过拟合**。
   >
   > 例如：
   >
   > ```
   > 训练集损失：0.01
   > 训练集准确率：99%
   > 验证集准确率：75%
   > ```
   >
   > 这说明模型可能记住了训练数据中的细节甚至噪声，却没有真正学到可以推广到新数据的规律。
   >
   > 常见原因包括：
   >
   > 1. **模型过于复杂**
   >     参数很多，而训练数据太少。
   > 2. **训练时间过长**
   >     后期模型开始记忆训练集。
   > 3. **训练集和测试集分布不同**
   >     例如训练集都是清晰图片，测试集却比较模糊。
   > 4. **数据存在泄漏**
   >     模型意外看到了与答案相关的信息。
   > 5. **损失函数与实际目标不完全一致**
   >     损失下降，不一定代表用户真正关心的指标提高。
   > 6. **类别不平衡**
   >     例如 95% 的样本都是类别 A。模型全部预测为 A，也有 95% 准确率，但对类别 B 完全无效。
   >
   > 因此，训练时不能只看训练损失，还要同时观察：
   >
   > - 验证损失；
   > - 验证集准确率；
   > - Precision、Recall、F1；
   > - 测试集表现。
   >
   > 典型的过拟合现象是：
   >
   > ```
   > 训练损失继续下降
   > 验证损失却开始上升
   > ```

7. StandardScaler 为什么只能在训练集上 fit？

   > `StandardScaler` 的作用是对数据进行标准化：
   >
   > x′=x−μσx' = \frac{x-\mu}{\sigma}x′=σx−μ
   >
   > 其中：
   >
   > - μ\muμ：均值；
   > - σ\sigmaσ：标准差。
   >
   > 调用：
   >
   > ```
   > scaler.fit(X_train)
   > ```
   >
   > 就是根据数据计算均值和标准差。
   >
   > 正确写法是：
   >
   > ```
   > scaler.fit(X_train)
   > 
   > X_train_scaled = scaler.transform(X_train)
   > X_test_scaled = scaler.transform(X_test)
   > ```
   >
   > 或者：
   >
   > ```
   > X_train_scaled = scaler.fit_transform(X_train)
   > X_test_scaled = scaler.transform(X_test)
   > ```
   >
   > 不能这样做：
   >
   > ```
   > scaler.fit(X_test)
   > ```
   >
   > 也不应该这样做：
   >
   > ```
   > scaler.fit(np.concatenate([X_train, X_test]))
   > ```
   >
   > 因为测试集代表未来尚未见过的数据。假如用测试集计算均值和标准差，就相当于训练过程提前获得了测试集的信息，这叫作：
   >
   > > 数据泄漏 Data Leakage
   >
   > 即使没有使用测试集标签，测试集的整体分布也已经泄漏给了模型。
   >
   > 实际部署时，遇到一个新样本，也必须使用训练阶段保存下来的均值和标准差进行处理：
   >
   > ```
   > 训练集计算出的 μ 和 σ
   >          ↓
   > 训练集、验证集、测试集、新数据全部使用这一套数值
   > ```
   >
   > 因此，可以总结为：
   >
   > > `fit()` 负责学习预处理参数，只能在训练集上进行；`transform()` 负责使用这些参数，可以应用于训练集、验证集和测试集。

8. scikit-learn 的 `fit()` 与 PyTorch 手写训练循环有什么联系？

   > 它们本质上做的是同一件事：
   >
   > > 根据训练数据不断调整模型参数，使模型的预测误差变小。
   >
   > 只是 scikit-learn 把训练过程封装在了 `fit()` 内部，而 PyTorch 通常需要自己写出来。
   >
   > #### scikit-learn
   >
   > ```
   > model.fit(X_train, y_train)
   > ```
   >
   > 这一行内部可能完成了：
   >
   > ```
   > 初始化参数
   > → 前向计算
   > → 计算损失
   > → 求梯度
   > → 更新参数
   > → 重复迭代
   > ```
   >
   > 这些过程被库封装起来了，所以用户看不到完整细节。
   >
   > #### PyTorch 手写训练循环
   >
   > ```
   > for epoch in range(epochs):
   >     for images, labels in train_loader:
   >         optimizer.zero_grad()
   > 
   >         outputs = model(images)
   > 
   >         loss = criterion(outputs, labels)
   > 
   >         loss.backward()
   > 
   >         optimizer.step()
   > ```
   >
   > 分别对应：
   >
   > | PyTorch 代码                 | 作用                 |
   > | ---------------------------- | -------------------- |
   > | `model(images)`              | 前向传播，计算预测值 |
   > | `criterion(outputs, labels)` | 计算损失             |
   > | `optimizer.zero_grad()`      | 清空上一轮梯度       |
   > | `loss.backward()`            | 反向传播，计算梯度   |
   > | `optimizer.step()`           | 根据梯度更新参数     |
   > | `for epoch...`               | 多轮重复训练         |
   >
   > 因此，可以把 scikit-learn 的：
   >
   > ```
   > model.fit(X_train, y_train)
   > ```
   >
   > 大致理解为把整个 PyTorch 训练循环封装成了一个方法。
   >
   > 不过不同模型的 `fit()` 内部算法并不完全一样：
   >
   > - 线性回归可能直接使用数学公式求解；
   > - 决策树通过不断选择最佳划分特征来训练；
   > - 逻辑回归可能使用梯度优化；
   > - KNN 的 `fit()` 主要是保存训练数据；
   > - 神经网络通常通过前向传播和反向传播训练。
   >
   > 所以更准确地说：
   >
   > > `fit()` 表示“让模型根据训练数据学习”，但内部不一定都采用神经网络的反向传播方式。
